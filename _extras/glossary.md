# Additional Glossary Entries

These terms appear in the glossary but are not currently referenced in any lesson.
They are retained here for reference and for possible use in future modules.

**90-9-1 rule**: 90% of people will watch, 9% will speak up, and 1% will actually do things.

**backfire effect**: The tendency for people to become more confident in a belief when
presented with evidence that contradicts it, because the contradiction threatens group
membership rather than simply updating information.

**benchmark**: A program (or set of programs) used to measure or compare the performance
of a piece of software.

**benevolent dictator**: A person who has final say on all aspects of a project's operation
and direction.

**build manager**: A program that keeps track of how files depend on one another and runs
commands to update any files that are out of date.

**CC-0** / **CC-BY** / **CC-BY-NC** / **Creative Commons**: Creative Commons licenses
vary in how much they restrict reuse. CC-0 places work in the public domain. CC-BY
allows any use with attribution. CC-BY-NC additionally prohibits commercial use.

**chief programmer**: A person responsible for most of the new code in a project, similar
to a chief surgeon in a hospital.

**chunking**: The act of grouping related concepts together so that they can be stored
and processed as a single unit.

**cognitive load**: The amount of working memory needed to accomplish a set of simultaneous
tasks.

**competent practitioner**: Someone who can do normal tasks in some domain with normal
effort under normal circumstances.

**concept map**: A box-and-connector diagram of the major elements in someone's mental
model of something.

**decorator**: A function that takes some other function as input and replaces it with
a modified version.

**design by contract**: A style of designing software in which functions specify the
pre-conditions that must be true in order for them to run and the post-conditions they
guarantee will be true when they return.

**design pattern**: A recurring pattern in software design that is specific enough to be
worth naming, but not so specific that a single best implementation can be provided by
a library.

**dynamic loading**: Importing a module into memory while a program is already running.

**entity-relationship diagram**: A diagram showing the tables in a database, the fields
in each, and the relationships between those tables' keys.

**expert**: Someone who can diagnose and handle unusual situations, knows when the usual
rules do not apply, and tends to recognize solutions rather than reasoning to them.

**expert blind spot**: The inability of experts to empathize with novices who are
encountering concepts or practices for the first time because they have forgotten what
it's like to *not* know something.

**false beginner**: Someone who doesn't yet know much about this domain in specific,
but who can transfer a lot of understanding from another domain.

**fixture**: The thing on which a test is run, such as the parameters to the function
being tested or the file being processed.

**flowchart**: A block-and-arrow diagram showing the orders in which statements in a
function can execute.

**Hippocratic License**: A software license that allows people to use and share software
so long as they do not violate human rights.

**information hiding**: Keeping the implementation details of a class or module internal
to that class or module, which allows those details to be changed without affecting other
classes or modules.

**JSON (JavaScript Object Notation)**: A way to represent data by combining basic values
like numbers and character strings in lists and dictionaries.

**lazy evaluation**: Delaying a computation until the value is actually needed.

**legacy code**: Software that is either no longer supported or very difficult to support
because of the accumulation of features and changes.

**logging**: Recording information about the execution of a program in a structured way
for later analysis.

**long-term memory**: The part of memory that stores information for long periods of time.
Long-term memory is large, but slow, and cannot be accessed directly.

**loose coupling**: Components in a software system are loosely coupled if they are
relatively independent, so that any one of them can be changed or replaced without
others having to be altered.

**markdown**: A markup language with a simple syntax intended as an alternative to HTML.

**mental model**: A simplified representation of the key elements and relationships of
some problem domain that is good enough to support problem solving.

**meritocracy**: A system in which advancement is theoretically based on individual
ability. In practice, meritocracies reward those who know the unwritten rules or can
afford to follow them.

**moral licensing**: The tendency for people who believe a system is already fair to
feel less need to monitor their own judgments for bias.

**motivated reasoning**: The tendency to evaluate arguments more critically when they
lead to conclusions we dislike than when they support conclusions we already hold.

**novice**: Someone who has not yet built a usable mental model of a domain and therefore
struggles with even basic tasks.

**ORCID (Open Researcher and Contributor ID)**: An alphanumeric code that uniquely
identifies someone publishing research results.

**paywall**: A feature that prevents people from accessing content unless they have paid
to do so.

**pipe**: Using the output of one computation as the input for the next, or the connection
between the two computations responsible for the data transfer.

**post-condition**: Something that is guaranteed to be true after a function runs
successfully.

**pre-condition**: Something that must be true before a function runs in order for it to
work correctly.

**profiler**: A tool that builds a performance profile showing how much time is spent on
each line (or in each function) of a program.

**reproducible example (reprex)**: A small, self-contained example of a problem.

**short-term memory**: The part of memory that briefly stores small amounts of information
that can be directly accessed by consciousness.

**situational awareness**: Understanding and paying attention to what's happening in one's
immediate environment.

**static site generator (SSG)**: A program that compiles source files to create HTML pages
for a website rather than generating HTML on the fly.

**UML (Unified Modeling Language)**: A set of graphical notations for modeling
object-oriented systems.

**YAML**: A way to represent nested data using indentation rather than the parentheses
and commas of JSON, often used in configuration files.
