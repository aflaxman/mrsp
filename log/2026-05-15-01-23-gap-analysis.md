# Gap Analysis: BTT Old Source vs. MRSP

## Prompt

Scan Markdown files in `~/btt/old/src` and identify six important ideas in software project management not covered in the current MRSP project.

## Actions

- Ran `brief .` to get project overview.
- Listed directories in `~/btt/old/src` and `~/mrsp`.
- Read lesson headings from all old source chapters.
- Read current MRSP lesson files: `project`, `governance`, `requirements`, `hirefire`, `performance`.
- Read old source chapters: `conflict`, `ip`, `security`, `process`, `automation`, `methods`, `teams`.

## Result

Six important management topics present in BTT old source but not in current MRSP:

1. Development process (Agile/Scrum/Kanban, pair programming, feedback loops)
2. Direct conflict resolution (escalation paths, problem contributors)
3. Intellectual property and licensing (copyright, ownership, open-source licenses)
4. Security (threat models, social engineering, digital hygiene)
5. Automation and CI/CD (build systems, automated testing as a management tool)
6. Empirical research methods (how to evaluate whether process changes help)
